Expected data format

For a brand new class, a CSV containing only a name column is all that is necessary.
The name column should contain the names of every student taking the class.

Once students have been paired, they are recorded in subsequent weeks. When saving the data, the original class list should be overwritten, and used in all following weeks. The data can be saved in a OneDrive folder that is shared with all staff involved in delivering any live sessions.

Upload class list

  1. Click the Browse button
    • Navigate to a class list file on your hard disk
    • The only requirement is that student names are contained in a column called name.
  2. Three new elements appear below the upload button.
    1. Set week/session
    2. Toggle include/exclude (i.e. present/absent)
    3. Select students
  3. Week / session
    • The week number is incremented automatically, but you can change it if you need to.
    • The assumption is that you are adding a session to the most recent week.
      • This means that if you are at the start of a new week, you should increment the week number.
    • You can also add a session identifier
      • There is a drop down list of previous session names to help you keep consistent naming.
      • E.g. Session 1 / Session 2.
  4. Toggle include/exclude
    • By default, the selection process assumes all students are present in the class.
    • If only a few students are present, remove Everyone from the selection box, and then select the students that are present.
    • To remove individual students form a large cohort, click the INclude/EXclude selector toggle button, and then select the students that are absent.
  5. Class display
    • The class of all present students is displayed on the right.
    • There is also a class size counter, useful for confirming you have selected the correct number of students.
  6. Once you are happy with your selection, move onto the Pair tab.

Creating pair groups

  1. Use the numeric input to decide on the number of people per group (defaults to 2).

  2. Click Pair Up

  3. If, for any reason, the pairs are not adequate, click the button again to re-shuffle

  4. If you want to copy the pairs to the clipboard, click the Copy to clipboard button.

    • The pairs are copied in a format that is easy to paste into an email document, or to paste into a Teams chat message.
    Group 1
    - Name 1
    - Name 2

  5. Press the save button to write to csv

    • Usually, you will want to overwrite the same file you uploaded.

  6. In subsequent session, repeat overwriting the same file as in the last step

    • the record of previous pairs should accumulate.

Error messages

Error 1: Missing name column

Your uploaded file must contain a column named name that lists all student names. This column should contain:

  • Individual student names for Week 0 (baseline class list)
  • Optionally, it may also contain paired names for subsequent weeks (format: name1~~name2)

Error 2: Missing Week 0 data

Week 0 serves as your baseline class roster and must contain individual student names only. This data is used to:

  • Establish the complete list of students in your class
  • Generate new pairings for students who havenâ€™t been paired yet

Error 3: Individual names in pairing weeks

Weeks 1 and above should only contain paired student names using the format name1~~name2. Each row should represent one complete pair. Individual names in these weeks indicate incomplete pairing data.

Error 4: Paired names in Week 0

Week 0 must contain individual student names only - no paired names allowed. Use the name1~~name2 format only for weeks 1 and above where actual pairings occurred. This format is used to avoid the many characters that can appear in names across many languages.
